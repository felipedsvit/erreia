erreia · Kanban leve em Go

Go Templ HTMX SSE Postgres 15 MB

erreia é um sistema Kanban construído em Go com Templ + HTMX + SSE usando Postgres LISTEN/NOTIFY para atualizações em tempo real. Autenticação por sessão com Argon2id, avatares no MinIO e CSS escrito à mão. Binário único de 15 MB — abaixo da meta de 40 MB.

começar código fonte

sobre

O nome "erreia" vem de uma brincadeira com "errei" em português — um aceno honesto à natureza iterativa do desenvolvimento. O projeto nasceu do desejo de construir uma aplicação web funcional completa em Go, do zero, sem frameworks JavaScript no frontend.

funcionalidades

Quadro Kanban

Colunas customizáveis com drag & drop via HTMX e atualização SSE em tempo real.

Tempo real

Postgres LISTEN/NOTIFY + SSE — sem WebSocket, sem polling. Múltiplos usuários veem mudanças instantaneamente.

Auth por sessão

Argon2id para hash de senha, sessões seguras com HttpOnly cookies.

Avatares no MinIO

Upload e serve de imagens via armazenamento S3-compatível (MinIO).

Binário único

15 MB, CGO-free, estático. Ideal para deploy em VPS, containers ou edge.

CSS escrito à mão

Sem frameworks CSS. Leve, sem dependências, carregamento instantâneo.

stack

Linguagem Go 1.24+
Views Templ — componentes Go type-safe compilados para HTML
Frontend dinâmico HTMX — atributos HTML para interatividade sem JS
Tempo real SSE (Server-Sent Events) + Postgres LISTEN/NOTIFY
Banco Postgres 17 + pgx
Auth Sessão HttpOnly + Argon2id
Storage MinIO (S3-compatível)
Container Docker + Compose, multi-stage, CGO_ENABLED=0

Docker

O projeto inclui Docker multi-stage e docker-compose completos para development e produção.

docker-compose.yaml (desenvolvimento)

version: "3.9"
services:
  postgres:
    image: postgres:17-alpine
    environment:
      POSTGRES_DB: erreia
      POSTGRES_USER: erreia
      POSTGRES_PASSWORD: erreia
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data

  minio:
    image: minio/minio
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
    ports:
      - "9000:9000"
      - "9001:9001"

  api:
    build: .
    ports:
      - "8080:8080"
    env_file: .env
    depends_on:
      - postgres
      - minio

volumes:
  pgdata:

docker-compose.prod.yaml (produção)

Para produção, use o arquivo docker-compose.prod.yaml que inclui:

  • Variáveis de ambiente de produção via .env.prod
  • Postgres com volume persistente e backup
  • MinIO com configuração de produção
  • Imagem multi-stage otimizada (~15 MB)

Dockerfile multi-stage

# Stage 1: Build
FROM golang:1.24-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 go build -o /app/server ./cmd/server

# Stage 2: Runtime
FROM alpine:3.20
RUN apk add --no-cache ca-certificates
COPY --from=builder /app/server /server
EXPOSE 8080
CMD ["/server"]

quickstart

Pré-requisitos: Go 1.24+, Docker, Docker Compose.

1. Clone e configure

git clone https://github.com/felipedsvit/erreia.git
cd erreia
cp .env.example .env

2. Suba os serviços

docker compose up -d postgres minio
# Aguarde os serviços ficarem prontos

3. Rode as migrações e inicie

go run ./cmd/server

4. Acesse

# Abra no navegador:
http://localhost:8080

Com Docker completo (recomendado)

docker compose up --build

Produção

cp .env.prod.example .env.prod
# Edite .env.prod com suas configurações
docker compose -f docker-compose.prod.yaml up -d --build
